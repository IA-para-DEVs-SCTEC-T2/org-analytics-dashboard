Oportunidades de evolução

urgente 🔐

Ativar branch protection

Configurar branch protection rules na main exigindo ao menos 1 aprovação antes do merge — isso transforma o processo de revisão em aprendizado ativo de code review.

urgente 📋

Template de PR no .github

Criar um template com checklist guiado (tipo de mudança, como testar, commit convencional) ajuda os alunos a estruturar melhor suas contribuições sem precisar decorar a convenção.

importante ✍️

Aula prática de conventional commits

Com 31% de adoção, a turma está no caminho certo. Uma aula prática com commitlint configurado no repositório pode consolidar esse hábito de forma definitiva para todos os 38 membros.

importante 📖

README em todos os repositórios

Vários repositórios ainda não têm descrição ou README. Um template simples (propósito, como rodar, como contribuir) pode ser adotado como requisito mínimo para as próximas atividades.

melhoria 🏆

Gamificar contribuições

Com 12 alunos ainda sem commits, um sistema leve de pontos ou badges (primeiro commit, primeiro PR revisado, primeiro conventional commit) pode ser um estímulo divertido para a participação.

melhoria ⚙️

CI básico nas atividades

Adicionar um workflow simples (lint + testes) no GitHub Actions introduz os alunos a CI/CD de forma prática. O grupo07 já usa testes — pode ser o modelo para os demais times.
